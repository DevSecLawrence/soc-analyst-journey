# Day 26 — GitHub Portfolio: Making My Work Recruiter-Visible

**Date:** 2026-06-10

---

## What I Did

Rewrote the main README for the soc-analyst-journey repo and created READMEs for 3 standalone showcase repos:

- `sigma-rules-collection` — Sigma detection rules with full documentation
- `yara-rules-collection` — YARA rules with testing methodology and known limitations
- `incident-reports` — phishing and Sysmon investigation write-ups with IOC tables

The main README went from "here's what this repo contains" to "here's who I am and here's what I've actually built." Big difference.

---

## What I Concluded

A GitHub repo without a good README is invisible. Nobody is going to click through 25 folders to figure out what you've done — they're going to read the landing page for 30 seconds and leave. The README is the pitch. Everything else is the evidence.

The showcase repos matter because they're linkable. When someone asks "do you have experience with Sigma?" I can send them a link to a repo that has nothing but Sigma rules, properly documented, with ATT&CK mappings and false positive notes. That's a completely different answer than "yes I've written some rules."

A CV says you can do something. A GitHub repo proves it.

---

## Assumption I Made

I assumed recruiters read code. Most don't — especially HR and non-technical recruiters. They read READMEs, they look at commit history to see if the work is real and consistent, and they look at whether the repo looks organised or chaotic. The actual code or rule content matters more to technical hiring managers who review your profile after the initial screen.

That means the README has to work for both audiences — non-technical enough that a recruiter understands what they're looking at, technical enough that a SOC manager sees real substance.

---

## Uncertainty I Have

I don't know how much weight GitHub actually gets in hiring decisions for entry-level SOC roles. Some job postings don't mention portfolios at all. I don't know if most hiring managers even look at GitHub links in applications or whether they go straight to certifications and years of experience. What I do know is that having it can only help — nobody's ever been rejected for having a well-documented portfolio.
