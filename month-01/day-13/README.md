# Day 13 — Alerts & Dashboards

This day covers creating a basic Splunk alert (medium) and building an investigation dashboard with scheduled reports/alerts (hard).

Files
- Alerts walkthrough: [day13-alerts.md](day13-alerts.md)
- Dashboard walkthrough: [day13-dashboard.md](day13-dashboard.md)
- Screenshots: [screenshots](screenshots/)

Quick start
1. Open Splunk Web: `http://localhost:8000` and sign in.
2. Follow `day13-alerts.md` to create a Scheduled alert that triggers on failed SSH login attempts.
3. Follow `day13-dashboard.md` to build a dashboard with panels for failed logins, sudo usage, and event volume.

Place screenshots under `screenshots/` named `screenshot-01.png`, `screenshot-02.png`, etc.
